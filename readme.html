About Portal

Portal was developed to provide easy access to regularly used tools. It has grown over the past couple of years from a small toolset, to a potentially company-wide intranet site.

-Topher Tebow, Lead Developer


Roadmap

Site Notifications The notifications table has been created in the database, but the site files have not yet been configured to create the notifications (within the Admin page - the panel is already in place with a "coming soon" placeholder), or display notifications on the site.
Move Query Results to Modals Currently, query results are displayed differently on the screen, depending on the extension. All query results will be modified to display in a modal to increase consistency across the site.
Google Safebrowsing API The current API is being retired in October of 2018. This will be updated to use the new API, prior to the current API being fully retired.

System Requirements

  • Apache 2.4 or higher
  • PHP 7.0 or higher
  • MySQL 5.7 or higher
  • Bootstrap 3.3.7
  • Font Awesome 4.7
  • JavaScript
  • jQuery 3.1.1 or higher
  • AJAX

Pages

Portal contains three groups of pages: landing, internal, and external.

Landing pages are department specific.

Internal pages may be restricted by department, and will not show if the logged in user does not have the proper permissions level. These pages contain tools that are useful to the assigned departments, but are not included in the departmental landing page.

External pages are any common tools or applications that are not a part of Portal. These will have menu links, in either the top menu or sidebar, but will open in a new tab. These menu options can be updated within the Admin page, and are maintained in the site's database.
